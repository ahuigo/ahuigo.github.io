# train llm
比如train a classifier based on qwen/deepseek, 两种方法：
- 全参数微调：直接在预训练模模的基础上，继续训练所有参数。这种方法需要大量的计算资源和时间，但可以获得较好的性能。
- LoRA = Low-Rank Adaptation（低秩自适应），是目前大模型微调的绝对主流技术。它的核心思想是：完全冻结预训练好的大模型（基模），只训练两个极小的 "补丁矩阵"(可能几MB)，就能让大模型学会特定任务。
    - 7B 模型微调仅需10GB 显存（QLoRA）,全参数微调则需要 60+GB 显存。

Note: 互补
QLoRA = LoRA + 4-bit 量化：极致压缩显存, 让消费级显卡能跑 7B/14B 模型
AdaLoRA = LoRA + 自适应秩分配：极致提升参数效率，用更少的参数获得更好的效果("解决 LoRA 的一刀切问题")

# QLoRA过程
1. 量化：将预训练模型的权重从32位浮点数（FP32）量化为更低位数的格式，如4位或8位。这可以显著减少模型的内存占用和计算需求。
2. LoRA微调：在量化后的模型基础上，使用LoRA方法进行微调。LoRA通过引入低秩矩阵来适应特定任务
3. 反量化：在推理阶段，将微调后的模型权重从量化格式转换回更高位数的格式，以获得更好的性能。
# 训练数据
- 训练数据的质量和多样性对模型性能有很大影响。通常需要大量的标注数据来训练一个高性能的模型。
- 数据增强：通过对训练数据进行各种变换（如旋转、缩放、翻转等）来增加数据的多样性，从而提高模型的泛化能力。
- 数据清洗：去除重复、错误或不相关的数据，以提高训练数据的质量。
- 数据划分：将数据划分为训练集、验证集和测试集，以评估模型的性能和防止过拟合。
# 训练过程
- 超参数调优：调整学习率、批量大小、优化器等超参数，以获得最佳的训练效果。
- 监控训练过程：使用工具如TensorBoard来监控训练过程中的损失、准确率等指标，以便及时调整训练策略。
- 模型保存和加载：定期保存模型的权重，以便在训练过程中恢复或在训练完成后进行推理。
- 评估模型性能：使用验证集和测试集来评估模型的性能，计算指标如准确率、精确率、召回率等，以了解模型的表现。 
# 训练资源
- 训练大型模型通常需要大量的计算资源，如GPU或TPU。可以使用云服务提供商（如AWS、Google Cloud、Azure）提供的计算资源，或者使用本地的高性能计算设备。
- 训练时间：根据模型的大小和训练数据的数量，训练过程可能需要数小时到数天不等。可以使用分布式训练来加速训练过程。
- 成本：训练大型模型可能会产生较高的计算成本，尤其是在使用云服务时。需要合理规划训练资源和时间，以控制成本
